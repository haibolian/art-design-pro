# art-design-pro Agent Guide

## 项目目标

- 当前项目为 `Vue 3 + TypeScript + Vite + Element Plus` 前端工程。
- 当前改造目标是适配若依后端能力，参考项目位于 `/home/haibo/Code/Vue3/RuoYi-Vue3`。
- 参考范围仅限业务逻辑、接口约定、数据结构、字典、权限、菜单等领域模型。
- 不参考若依前端项目中的技术栈选型、工程化方案、旧式组件封装和页面实现方式。

## 当前阶段

- 权限与菜单改造已完成。
- 暂不批量迁移业务页面。
- 当前优先级是先搭建基础设施。
- 近期核心目标包括：
  - 字典能力抽象
  - 面向配置的 `pro-table` 基础设施
  - 查询、分页、列配置等表格通用能力沉淀

## 工程原则

- 默认保持现有项目技术栈：`Vue 3`、`Composition API`、`TypeScript`、`Vite`、`Element Plus`。
- 新增基础设施优先做类型收敛、职责分层、接口清晰，不为兼容旧实现引入隐式兜底。
- 不新增静默降级、mock 成功路径、吞错逻辑；失败应明确暴露，方便调试。
- 优先抽象可复用能力，不提前迁移具体业务页。
- 设计上优先“配置驱动 + 类型约束 + 可扩展插槽/钩子”，避免写死业务分支。

## 目录感知

- `src/api`: 接口请求定义
- `src/types`: 全局类型、接口模型、组件类型
- `src/utils/http`: HTTP 基础能力
- `src/utils/table`: 表格相关工具
- `src/components`: 通用组件与基础组件
- `src/store`: 全局状态
- `src/views/system`: 系统管理相关页面，可作为若依领域能力对齐时的落点参考

## 协作约定

- 修改前先阅读现有实现，尽量沿用仓库已有模式与命名。
- 若涉及若依对齐，先确认字段、入参、出参与字典语义，再决定前端抽象。
- 优先补基础设施与类型，不随意引入页面级耦合。
- 未被明确要求时，不主动编写与当前目标无关的页面迁移代码。

## 常用命令

- 安装依赖：`pnpm install`
- 本地开发：`pnpm dev`
- 生产构建：`pnpm build`
- 代码检查：`pnpm lint`
- 自动修复：`pnpm fix`
